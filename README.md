# Cpp_SnakeGame

1. 맵(Map) 및 스테이지 구성
   
   
2차원 배열 기반 맵: 최소 21x21 크기의 2차원 배열로 게임 화면을 구성해야 합니다

벽 구분: 일반 벽(Wall)과 게이트로 변할 수 없는 Immune Wall을 명확히 구분하여 데이터로 관리하고 화면에 표시해야 합니다

다중 스테이지: 서로 다른 맵 데이터로 구성된 최소 4개 이상의 스테이지를 구현해야 하며, 미션 달성 시 다음 스테이지로 전환되어야 합니다



2. Snake 제어 및 물리 규칙
   
   
이동 및 입력: 방향키를 입력받아 Snake가 이동하며, 진행 방향의 반대 방향으로의 이동은 금지됩니다(입력 시 게임 오버)

틱(Tick) 시스템: Snake는 일정 시간(예: 0.5초 또는 1초)마다 자동으로 한 칸씩 이동해야 합니다

충돌 판정: Snake가 자신의 몸통(Body)이나 벽(Wall)에 충돌하면 실패(Game Over) 처리됩니다

(추가 사항) snake가 움직이는 일정 시간은 논의 후 결정합니다


3. 아이템 시스템
   
   
Growth & Poison Item:

-Growth Item: 획득 시 몸의 길이가 1 증가합니다

-Poison Item: 획득 시 몸의 길이가 1 감소하며, 길이가 3보다 작아지면 실패합니다

출현 규칙: 아이템은 Snake의 몸이 없는 임의의 위치에 나타나야 하며, 동시에 최대 3개까지만 존재할 수 있습니다

시간 제한: 아이템은 출현 후 일정 시간(예: 5초)이 지나면 사라지고 다른 위치에 다시 나타나야 합니다

(추가 사항) 새로운 item에 대한 아이디어가 있으면 논의 후 추가


4. 게이트 시스템
   
   
출현 및 진입: 두 개가 한 쌍인 게이트가 벽의 임의 위치에 나타납니다. Snake가 한 게이트로 들어가면 다른 게이트로 나옵니다

진출 방향 결정:

-가장자리 벽: 항상 맵 안쪽 방향으로 진출합니다

-맵 내부 벽: 진입 방향과 일치하는 방향을 우선으로 하며, 불가능할 경우 시계방향 -> 반시계방향 -> 반대방향 순으로 결정합니다

특수 규칙: Snake가 게이트를 통과 중일 때는 게이트가 사라지거나 위치가 바뀌지 않습니다


5. 점수 및 미션 시스템
   
   
Score Board: 우측 화면에 현재/최대 길이, 획득한 아이템 수, 게이트 사용 횟수, 경과 시간 등을 실시간으로 표시해야 합니다

Mission Board: 각 항목별 목표치를 표시하고, 이를 모두 달성(v 표시 등)하면 해당 스테이지가 클리어되도록 구현합니다
